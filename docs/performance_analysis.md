## Текущие проблемы производительности

### Вызов подпроцесса при каждом изменении ползунка
**Суть:** Метод `applyGamma()` запускает подпроцесс для выполнения команды xgamma при каждом изменении ползунка.

**Влияние:** 
- Задержка около 50мс на каждый вызов
- При быстром перемещении ползунка может накапливаться очередь вызовов

**План:**
- Добавить throttling для вызовов `applyGamma()` аналогично обновлению изображения
- Использовать QTimer с задержкой 50-100мс для отложенного применения гаммы
- Это позволит применять гамму только после того, как пользователь прекратил перемещать ползунок
